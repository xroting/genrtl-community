# genrtl-community

Public community hub for genRTL questions, troubleshooting, feature requests,
workflow discussions, and reusable IP requests.

## Use GitHub Discussions First

Use GitHub Discussions for:

- questions about how to use genRTL
- troubleshooting and reproducible failures
- feature requests and product feedback
- CBB or reusable IP requests
- assertion and verification workflow discussions
- community showcases and polls

Use GitHub Issues when a discussion has already converged into a concrete,
actionable bug or implementation task.

Do not post private RTL, credentials, billing details, customer data, or
security-sensitive information in public discussions.

## Discussion Categories

The repository discussion space is organized around these categories:

- `Announcements`: official updates, release notes, and maintenance notices
- `Questions`: how-to questions and workflow guidance
- `Troubleshooting`: reproducible failures, logs, and environment problems
- `Feature Requests`: product and UX proposals
- `CBB Requests`: requests for reusable IP, modules, or interfaces
- `Assertion Verification`: assertions, bind flow, TB scaffold, and debug flow
- `Show and Tell`: examples, demos, and usage reports
- `Polls`: roadmap and prioritization votes

## How To Get Faster Help

When opening a discussion, include:

- the genRTL version or extension version, if known
- your OS and runtime environment
- what you tried already
- the expected result
- the actual behavior
- a minimal reproducible example when possible
- sanitized logs or snippets that do not expose private data

## Triage Workflow

- Maintainers may relabel or recategorize discussions for clarity.
- Questions and troubleshooting threads should end with a marked answer when
  resolved.
- Mature feature or CBB requests may be converted into GitHub Issues for
  execution tracking.
- Off-topic, duplicate, or unsafe content may be locked or removed.
