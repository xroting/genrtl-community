# Security Policy

## Public Discussions Are Not For Sensitive Reports

Do not use GitHub Discussions, Issues, or public comments to report:

- security vulnerabilities
- private RTL or internal design details
- credentials, tokens, or secrets
- billing or account-sensitive information
- customer or partner data

## How To Report A Security Issue

Use a private reporting path:

- GitHub private vulnerability reporting, if enabled for this repository
- otherwise, contact the repository owner or maintainer through a private
  GitHub channel

Include:

- a clear description of the issue
- impact and affected area
- steps to reproduce
- any suggested mitigations

## Response Expectations

Maintainers will review the report, assess severity, and coordinate remediation
before requesting any public disclosure.
