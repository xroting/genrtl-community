# Contributing

## Where To Start

This repository is primarily for community support and product feedback.

Use GitHub Discussions for:

- `Questions`
- `Troubleshooting`
- `Feature Requests`
- `CBB Requests`
- `Assertion Verification`
- `Show and Tell`
- `Polls`

Use GitHub Issues only after a discussion has become a clear implementation
task, bug, or maintenance item.

## Before Opening A Discussion

- Search existing discussions first.
- Pick the most specific category available.
- Use the provided form when the category has one.
- Remove private RTL, secrets, billing data, and customer information.

## What Good Discussions Include

- a clear problem statement
- the goal or intended outcome
- what you already tried
- version and environment details when relevant
- a minimal reproducible example when relevant
- sanitized logs or snippets

## Maintainer Triage

Maintainers may:

- change the category or title
- apply shared labels
- ask for more information
- mark an answer for resolved Q&A threads
- convert mature discussions into issues
- lock or remove unsafe, duplicate, or off-topic threads

## Shared Labels

Discussions, issues, and pull requests can share the same label set. The
recommended baseline is documented in
[.github/DISCUSSION_LABELS.md](.github/DISCUSSION_LABELS.md).

## Security And Sensitive Data

Do not disclose vulnerabilities, private code, credentials, or customer data in
public discussions. Use the private path described in [SECURITY.md](SECURITY.md).
